# Epic 8 Retrospective - Drag-and-drop Lesson Rescheduling

Status: done

## Epic Summary

- Epic: 8
- Stories: 1 total / 1 done
- Focus: weekly timetable drag-and-drop lesson time rescheduling

## What Went Well

1. Existing lesson patch API를 재사용해서 서버 변경 없이 기능을 빠르게 추가할 수 있었다.
2. 드래그 동작 이후 클릭 오동작을 억제해 편집 모달 충돌 없이 안정적으로 동작했다.
3. 드래그 이동 시 수업 길이 보존 로직을 명확히 분리해 회귀 위험을 낮췄다.
4. E2E에 드래그 시나리오를 추가해 실제 사용자 경로를 자동 검증했다.

## What Did Not Go Well

1. 현재 드롭 단위가 `HH:00` 슬롯 고정이라 30분 단위 이동 요구에는 바로 대응되지 않는다.
2. 다중 카드 겹침/충돌 시각 피드백은 기본 수준으로, 고급 편집 UX는 아직 부족하다.

## Key Lessons Learned

1. 스케줄 변경 기능은 API 변경보다 클라이언트 상호작용 안정화가 핵심 리스크였다.
2. DnD 기능은 클릭/드래그 이벤트 경합 제어를 초기에 설계해야 유지보수가 쉽다.
3. 시각 피드백(드롭 타겟/드래그 상태)은 기능 완성도와 사용자 신뢰에 직접 영향을 준다.
